\section{Reproducibility}

The rescue package binds headline results to a canonical JSON evidence registry. Each source report is identified by a project-relative path and SHA-256. The registry records the dataset version, partition, evidence class, metric contract, seed count, headline values, confidence intervals, and limitations used in the generated paper tables; it is not the source of every implementation or resource parameter. Paper tables are generated from that registry rather than transcribed manually.

\subsection{Protocol and resource provenance}

Method and compute numerics are bound separately to frozen configurations, source modules, and run artifacts. For Loop191, \texttt{configs/loop191/c0\_clean\_v3\_imp\_control\_pilot15.yaml} fixes the $384\times384$ input, MiT-B3/SMP model, 15 epochs, batch size 32, and learning rate $6\times10^{-5}$; its inherited preprocessing configuration fixes CLAHE (2.0; $8\times8$), percentile stretch (1, 99), and median kernel 3. \texttt{src/lesion\_robustness/postprocessing.py} supplies the inherited default postprocessing: threshold 0.5, closing kernel 5, hole filling, minimum area ratio 0.005, and largest-component retention. For Loop192, \texttt{configs/loop192/nnunet\_v2\_clean\_v3\_raw\_100ep.yaml} fixes 2,008 training rows, 100 epochs, raw-RGB $256\times256$ inputs, and the $384\times384$ corruption and metric canvases.

Loop205 regional-forest settings and five group-disjoint folds are defined in \texttt{src/lesion\_robustness/loop205\_protocol.py}: two SLIC scales, 96 trees, maximum depth 12, minimum leaf size 3, feature fraction 0.75, and sample fraction 0.65. Loop206 contour constants are defined in \texttt{src/lesion\_robustness/loop206\_active\_contour.py}; \texttt{confirmatory384\_screen.json} records the selected \texttt{neutral\_mid\_30\_s2} policy with $\alpha=12$, saliency weight 0.35, probability range 0.01--0.75, 30 iterations, smoothing 2, zero balloon force, 0.05 threshold offset, and area bounds 0.20--2.00. The six frozen control/candidate seed configurations under \texttt{configs/loop206/}, \texttt{pilot\_preflight.json}, and \texttt{current\_bdou\_loop206\_student\_pilot\_final\_report.json} collectively establish 20 epochs, seeds 206/1206/2206, 308 fit groups, 76 holdout groups, three corruptions, and 10,000 paired cluster-bootstrap resamples. The final report supplies the recorded 2,239.7-second stage time, 15.644 GiB GPU peak, 8.612 GiB process-group resident-memory peak, and 0.018 GiB swap increase; \texttt{pilot\_preflight.json} identifies the RTX 5060 Ti and records 15.928 GiB total GPU memory.

The repository includes the Loop191, Loop192, and Loop206 configs; model constructors; preprocessing and postprocessing implementations; metric code; Loop205 regional saliency protocol; Loop206 active-contour implementation; evidence builder; table builder; tests; and website source. Runtime data, checkpoints, probability caches, and large artifacts remain outside Git. The selected Loop206 control and candidate checkpoints are verified by SHA-256 at application startup.

For arbitrary uploaded images, the candidate model requires a deployment saliency prior fitted only from the 308 Loop206 fit groups. Deployment is authorized only if the resulting contour generator reproduces all 76 frozen holdout clean contours byte-for-byte. If data identity, source hashes, model hashes, or contour parity fail, candidate upload inference remains disabled rather than using an approximate channel.

The recorded hardware is an RTX 5060 Ti workstation with Python 3.12. Environment dependencies are locked with \texttt{uv}. A second Windows laptop performs clean-clone CPU tests and LaTeX compilation without datasets or weights. The final release records test, paper-audit, PDF-build, GPU-smoke, and public-tunnel receipts.
