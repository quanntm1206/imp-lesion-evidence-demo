\section{Reproducibility}

The rescue package binds headline results to a canonical JSON evidence registry. Each source report is identified by a project-relative path and SHA-256. The registry records the dataset version, partition, evidence class, metric contract, seed count, headline values, confidence intervals, and limitations used in the generated paper tables; it is not the source of every implementation or resource parameter. Paper tables are generated from that registry rather than transcribed manually. Registry-only verification establishes semantic and hash consistency, not full experiment reproduction.

\subsection{Protocol and resource provenance}

The source reports used for Loop191 and Loop192 record the $384\times384$ IMP setup, the raw-RGB $256\times256$ nnU-Net setup, and the historical $384\times384$ metric canvas. However, the cited Loop191 and Loop192 configuration files, their implementation modules, the inherited postprocessing module, and the compact source reports are not tracked in this release. Their numeric settings are therefore source-report evidence available to the strict local audit, not a clone-runnable recipe supplied by this commit.

The tracked release does include the Loop205 regional-saliency module, the Loop206 active-contour module, and the seed-206 zero-channel control configuration. The other five Loop206 arm/seed configurations, preflight receipt, and final scientific report are not tracked in this release. The evidence registry binds the final report hash and headline deltas, while the operator-local report records 20 epochs, seeds 206/1206/2206, 308 fit groups, 76 holdout groups, three views, resource measurements, and the group-bootstrap definition. A fresh clone can validate these recorded claims and rebuild tables, but cannot re-run all six training jobs from tracked files alone.

The repository includes the demo model constructor, metric and evidence code, Loop205 regional saliency protocol, Loop206 active-contour implementation, one Loop206 control config, table builder, tests, and website source. Runtime data, additional experiment configs, checkpoints, probability caches, source reports, and large artifacts remain outside Git. The selected Loop206 control and candidate checkpoints are verified by SHA-256 at application startup.

For arbitrary uploaded images, the candidate model requires a deployment saliency prior fitted only from the 308 Loop206 fit groups. Deployment is authorized only if the resulting contour generator reproduces all 76 frozen holdout clean contours byte-for-byte. If data identity, source hashes, model hashes, or contour parity fail, candidate upload inference remains disabled rather than using an approximate channel.

The recorded experiment hardware is an RTX 5060 Ti workstation with Python 3.12. Environment dependencies are locked with \texttt{uv}. An operator log states that a same-machine clean-clone simulation passed the portable workflow, but no tracked, commit-bound receipt was retained; this release therefore treats that execution as unverified. Physical execution on the RTX 4060 laptop, including a laptop CUDA receipt, remains unverified; no physical laptop execution is claimed. Browser rendering and desktop/mobile screenshots also remain unverified.
